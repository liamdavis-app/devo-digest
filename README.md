# UK Devolution Digest

A daily email digest of UK local-government and fiscal-devolution news, reports
and research. Every morning it scans a curated set of think-tank, Parliament and
sector-press feeds, keeps anything matching your watchlist, asks Claude to
summarise each item, and emails you the result.

Each item gives you: **who it's from**, **a one-line blurb on that author/org**,
**a synthesis of what's in it**, **the publication date**, and **a link to the source**.

No servers, no app to maintain. It runs on GitHub Actions (free) and emails your inbox.

---

## What you get each morning

A briefing-style email designed for fast triage on a phone:

- Items **grouped by source type** — Official & Parliament, Think tanks & research,
  Sector press & other — each with a count.
- Each item leads with the **article title**, then a monospace reference line
  (author/org · date), a short blurb on who they are, and **one tight line** on
  what the piece says and why it matters — enough to decide whether to tap through.
- A coloured left rule keys each item to its source type (brass = official,
  teal = think tank, plum = press) so you can scan by category at a glance.
- **Native dark mode** (via `prefers-color-scheme`) — reads correctly in iOS Mail
  day or night.
- A **preheader** line summarises the day next to the subject in your inbox.

If nothing new matched, you get a short "nothing today" note instead, so you
always know the scan ran.

The design targets Apple Mail specifically (a modern, standards-friendly
renderer), which is why it can use real CSS rather than legacy table hacks.

## Setup (about 15 minutes, one-time)

### 1. Put these files in a GitHub repo
Create a new **private** repo and add all the files in this folder
(`digest.js`, `sources.js`, `package.json`, and `.github/workflows/digest.yml`).

### 2. Get an Anthropic API key
- Go to the Anthropic Console, create an API key.
- Add a few dollars of credit — this job costs roughly **a cent or two per day**
  (a handful of items summarised with Sonnet).

### 3. Set up email sending (SMTP)
Easiest option is a Gmail account with an **App Password**:
- Turn on 2-Step Verification on the Google account.
- Create an App Password (Google Account → Security → App passwords).
- Use `smtp.gmail.com`, port `465`, your address as the username, the app
  password as the password.

(Any SMTP provider works — Fastmail, a custom domain, SendGrid, etc.)

### 4. Add secrets to the repo
In the repo: **Settings → Secrets and variables → Actions → New repository secret.**
Add each of these:

| Secret name         | Value                                             |
|---------------------|---------------------------------------------------|
| `ANTHROPIC_API_KEY` | your Claude API key                               |
| `SMTP_HOST`         | e.g. `smtp.gmail.com`                             |
| `SMTP_PORT`         | e.g. `465`                                        |
| `SMTP_USER`         | the sending email address                         |
| `SMTP_PASS`         | the SMTP / app password                           |
| `DIGEST_TO`         | the inbox you want the digest sent to             |

### 5. Test it
- Go to the repo's **Actions** tab → **Daily Devolution Digest** → **Run workflow**.
- Watch the log. Within a minute or two you should get an email.
- After that it runs automatically every day at the cron time in `digest.yml`.

## Tuning it

- **Sources:** edit `SOURCES` in `sources.js`. Each row is `{ name, feed }` where
  `feed` is an RSS/Atom URL. Add or remove freely.
- **Topics:** edit `KEYWORDS` in `sources.js`. An item is kept if its title or
  summary contains any keyword. Add narrower terms to cut noise, broader ones to
  widen the net.
- **Time of day:** edit the `cron` line in `.github/workflows/digest.yml`
  (it's in UTC).
- **Lookback window:** `LOOKBACK_HOURS` in `sources.js` (default 26h).

## Two things to verify against current Anthropic docs
Because model names and API details change over time, confirm these once when
you set up (both in the Anthropic API docs):

1. **Model string** — `digest.js` uses `claude-sonnet-4-6`. Check this is the
   current Sonnet identifier; swap it if a newer one is out.
2. **API version header** — it sends `anthropic-version: 2023-06-01`, the
   long-standing stable value. Fine unless the docs tell you otherwise.

Everything else (the `/v1/messages` endpoint shape, `x-api-key` auth) is stable.

## Notes / honest caveats
- Summaries are generated by an LLM from the feed's own excerpt, not the full
  article. They're a triage layer, not a substitute for reading the source —
  the email says as much in its footer.
- RSS excerpts vary in richness. Some feeds give a full paragraph; some give a
  sentence. Thin excerpts produce thinner summaries — the prompt tells Claude to
  admit when it can't summarise confidently rather than invent.
- The Google News catch-all feed is broad; if it adds noise, tighten `KEYWORDS`
  or remove that source row.
- Feeds occasionally move or break. A dead feed is logged and skipped, it won't
  stop the run.
